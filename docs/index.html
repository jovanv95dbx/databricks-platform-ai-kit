Installation

Prerequisites, install options, flags, and verification

Prerequisites

  • Python >= 3.10
  • Terraform >= 1.9.0
  • Claude Code (Cursor or CLI)
  • Cloud CLI authenticated — az login, aws configure, or gcloud auth login
  • Databricks account ID from the accounts console

Install prerequisites

# Terraform (macOS)
brew install terraform

# Verify versions
terraform -v
python3 --version

Install options

Lightweight

Skills only

Just the AI skills. Claude writes all Terraform from scratch using skill knowledge and the shell. No extra Python dependencies.

./install.sh --skills-only
Recommended

Full install

Skills plus the core library and MCP server. Adds credential management (macOS Keychain) and Terraform run tracking.

./install.sh

The full install copies skills, installs databricks-platform-core and databricks-platform-mcp in editable mode, and registers the MCP server in your Claude Code settings.json.

Restart Claude Code after a full install to activate the MCP server.

Flags

Flag Description
--skills-only Install skills only — no MCP server, no Python dependencies
--global Install to ~/.claude/ (global) instead of ./.claude/ (project-level)
--force Overwrite existing skill installations without prompting
-h, --help Show help

Verify the installation

# Check skills were installed
ls .claude/skills/

# Check MCP server is registered (full install only)
cat .claude/settings.json

# Check Terraform is available
terraform -v

Uninstalling

# Remove skills
rm -rf .claude/skills/platform-provisioning
rm -rf .claude/skills/unity-catalog-setup
rm -rf .claude/skills/identity-governance
rm -rf .claude/skills/workspace-config
rm -rf .claude/skills/private-networking

# Uninstall Python packages (if full install was used)
pip uninstall databricks-platform-core databricks-platform-mcp

# Remove MCP server entry from .claude/settings.json manually

How it works

Describe what you need, Claude writes and deploys the Terraform
1
You describe what you need — e.g. "Set up 3 Databricks workspaces (dev/stg/prod) on Azure with Unity Catalog and proper groups"
2
Claude asks the right questions: cloud provider, region, environment strategy, naming conventions
3
Claude checks your auth, gathers inputs, and writes Terraform tailored to your request
4
Infrastructure gets deployed end to end via terraform init, plan, and apply
5
Claude offers to verify with parallel test notebooks (classic cluster, SQL warehouse, serverless)

Architecture

skills/                              # 5 focused AI skills
  platform-provisioning/
    SKILL.md                         #   Shared workflow + interaction guidance
    AZURE.md / AWS.md / GCP.md       #   Cloud-specific patterns + gotchas
  unity-catalog-setup/
    SKILL.md + AZURE.md / AWS.md / GCP.md
  identity-governance/
    SKILL.md                         #   Cloud-agnostic
  workspace-config/
    SKILL.md                         #   Cloud-agnostic
  private-networking/
    SKILL.md + AZURE.md / AWS.md

core/                                # Shared Python library (Terraform runner,
                                     #   credentials, account API, templates)
mcp-server/                          # Optional MCP server for enhanced UX
The MCP server is optional. The skills teach Claude the patterns. If the MCP server is running, Claude gets structured credential management and run tracking. If not, Claude uses your cloud CLI and Terraform via the shell — and it still works.
